## Overview

Every schema you expose to Supasheet gets a built-in SQL editor at `/$schema/sql-editor/$snippet`. It's a safe, read-only way to explore data and prototype queries without leaving the app — handy for support, debugging, and quick ad-hoc analysis.

## Key Features

- **Monaco-powered editor** — syntax highlighting, multi-cursor, find/replace
- **Schema-scoped** — every snippet belongs to a specific schema and runs against that schema's exposed views and tables
- **Saved snippets** — name a snippet and revisit it via the URL or sidebar
- **Result grid** — tabular result viewer with copy / export
- **Respects RLS** — queries run as the signed-in user; every row visible matches that user's permissions and policies

## Read-Only by Design

The editor only accepts `SELECT` statements (and other read-only constructs such as `WITH … SELECT`). Mutations and DDL are intentionally blocked:

| Allowed | Not allowed |
|---------|-------------|
| `SELECT ... FROM ...` | `INSERT`, `UPDATE`, `DELETE`, `TRUNCATE` |
| `WITH ... SELECT ...` | `CREATE`, `ALTER`, `DROP` |
| Joins, aggregates, set operations | `GRANT`, `REVOKE`, role changes |

For changes to data, use the auto-generated CRUD UI. For schema changes, use Supabase migrations.

<Callout type="warn">
Queries that take very long, return huge result sets, or run on un-indexed columns will affect everyone — they still execute on the database. Prefer pagination (`LIMIT`) and explicit `WHERE` clauses while exploring.
</Callout>

## Typical Workflows

### Data exploration

```sql
SELECT status, count(*) AS tasks
FROM desk.tasks
WHERE assignee_id = auth.uid()
GROUP BY status
ORDER BY tasks DESC;
```

### Prototyping a future report or chart view

Iterate on the query in the editor, then promote the final version into a view with the matching JSON comment (`{"type":"report"}` or `{"type":"chart"}`). See [Reports](/docs/guide/reports) and [Charts](/docs/guide/charts).

### Checking permissions

```sql
SELECT * FROM supasheet.get_permissions();      -- All permissions held by current user
SELECT * FROM supasheet.get_tables('desk');     -- Tables in 'desk' visible to current user
```

### Inspecting audit trail

```sql
SELECT created_at, operation, changed_fields
FROM supasheet.audit_logs
WHERE record_id = '...'
ORDER BY created_at DESC;
```
